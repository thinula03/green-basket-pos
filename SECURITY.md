# Security Policy

## Supported Versions

Security fixes are handled on the latest version of the project.

| Version | Supported |
| --- | --- |
| Latest `main` branch | Yes |
| Older forks or modified builds | No |

## Reporting a Security Issue

Please do not open a public GitHub issue for serious security problems.

If you find a security issue, contact the maintainer privately. If no private
contact is listed yet, open a GitHub issue with a general title such as
`Security contact request` and do not include exploit details.

Helpful information to include privately:

- What the issue is
- How it can be reproduced
- What files or features are affected
- Your operating system
- Suggested fix, if you have one

## Notes

This app is designed for offline local use. It stores data on the user's own
computer and does not include a remote server by default.
